## 一、函数的本质

```python
def 函数名(参数):
    """做某件事"""
    # 具体步骤
    return 结果
```

- `def` = define，**教会** Python 一个技能
- 函数本身不会执行，需要**调用**（写 `函数名()`）才真干活
- `(参数)` = 告诉使用者"你要用我，得给我材料"
- `return` = 把结果送回给调用处，同时函数结束

### 调用过程（最重要）

```python
第 1 行：result = call_ai_api(content)
                       ↓
           跳进 call_ai_api 函数里面执行
                       ↓
           执行到 return structured_data
                       ↓
           跳回第 1 行，结果存进 result
                       ↓
           继续往下走
```
就像看书时看到"见第 50 页注释"，翻过去看完再翻回来。

## 二、if-else 是二选一，不是先后做

```python
if not os.path.exists("raw_data"):
    os.makedirs("raw_data")
    exit()     ← 程序直接死透，else 永远不会执行
else:
    print("文件夹已存在")
```

关键理解：`if` 只问**一次**条件，真就走 `if`，假就走 `else`。
- 不存在 `if` 干完又去干 `else` 的情况
- `exit()` = 当场断电关机，不是"回到路口重新选"
- 觉得"两条路都走了"是因为**手动运行了两次脚本**

## 三、三组"易混"概念

### 3.1 `import` vs `from ... import`

| 写法 | 使用时 | 场景 |
|---|---|---|
| `import datetime` | `datetime.datetime.now()` | 全拿来，用得少时 |
| `from datetime import datetime` | `datetime.now()` | 只拿一个常用类，省前缀 |

### 3.2 全大写变量（常量）

```python
DATA_FILE = "work_log.json"
```
全大写 = Python 约定：**"这个值固定不变，不要在后面改它"**。好处：只写一次文件名/路径，后面反复用，想改名只改一处。

### 3.3 Python 字典 vs JSON 字符串

```
Python 字典（内存中）              JSON 字符串（硬盘上）
  {"项目": "AI播客"}    dump →     {"项目": "AI播客"}
                        load ←
  用 d["项目"] 取值                 不能取值，报错
  关掉脚本就消失                     文件还在，下次能读回来
```

- `json.dump(数据, 文件)` = 字典 → 写进文件
- `json.load(文件)` = 从文件 → 读回字典
- `json.dumps`（多一个 s）= 转成字符串展示，不存文件

## 四、错误处理 try-except

```python
try:
    with open("文件", "r") as f:
        return json.load(f)
except FileNotFoundError:
    return []   ← 文件不存在也别崩溃，给个空列表
```

- `try` = "试着做下面的事"
- `except 某错误类型` = "如果出了这种错，就这么处理"
- 不处理 → 程序崩溃红字报错；处理了 → 优雅继续跑
- 常见的错误类型：`FileNotFoundError`、`Timeout`、`JSONDecodeError`

## 八、读 daily_work_log.py 新学的知识点

### 8.1 `.strip()` 去空白
```python
"  你好  \n".strip()  →  "你好"
```
判断"空文件"时先 strip 再判断，否则全是空格/换行的文件不会被跳过。

### 8.2 变量名 vs 字符串 —— 引号决定一切

| 写法 | 意思 |
|---|---|
| `results` | **变量** — 去内存找这个名字的盒子 |
| `"results"` | **字符串** — 纯文本，就是字母 r-e-s-u-l-t-s |
| `f"内容：{变量}"` | **格式化** — `{}` 里的变量名替换成真实值 |

引号就是分界线：**带引号是文本，不带引号是变量。**

### 8.3 `f` 的两种身份

| 出现方式 | 意思 |
|---|---|
| `f"..."`（f 紧贴引号） | **格式化标记**，开启 `{}` 变量替换 |
| `f = open(...)`（f 后面空格） | **变量名**，只是个普通字母 |

只有贴着引号的 `f` 才是格式化，其他都是变量名。

### 8.4 打包的是 `{}`，不是变量名

```python
payload = {               # {} 才是真正造盒子的
    "model": MODEL_NAME,  # 往盒子里放东西
}
```

变量名（`payload`）只是贴在盒子上方便以后找它的标签。

### 8.5 Python 缩进规则

- 进了 `{}`、`:` 就要往右缩进
- 同一层必须对齐（空格数一致）
- 缩进不对直接报错，不会悄悄乱跑
- 编辑器会自动帮你缩进

### 8.6 API 各部分有固定位置，不能乱塞

| 部分 | 位置 | 类比 |
|---|---|---|
| `headers` | 信封 | 证件、内容格式 |
| `payload["model"]` | 请求体 | 用哪个模型 |
| `payload["messages"]` | 请求体 | 要处理的对话内容 |
| `payload["temperature"]` | 请求体 | 创造力大小 |

每个部分站在 API 设计者规定的位置上，放错位置服务器不认。

### 8.7 `glob` 通配符 `*` 的原理

`"raw_data/*.txt"` 执行过程：
1. 走进 `raw_data/` 文件夹
2. 扫一眼所有文件，逐个问"结尾是不是 .txt？"
3. 是 → 装进列表；不是 → 跳过
4. 返回列表

`*` = **"文件名中间随便叫什么都行"**。

### 8.8 `for` 循环的底层机制

```python
for f in all_files:   # 内部做了这些事：
```
1. 看列表有几个元素
2. 设隐形指针指向第 0 个
3. 取出指针指向的元素 → 赋给 `f`
4. 指针后移一位
5. 重复 3-4，直到越界 → 结束

### 8.9 `with open` 自动关门

```python
with open("文件", "r") as f:
    content = f.read()
# 缩进结束 → 自动 f.close()
```

省掉手动写 `f.close()`。

### 8.10 `return` 是"交还给调用者"

```python
def process_files():
    ...
    return all_results, output_paths   # 把两个篮子还给喊我的人

# 调用处：
results, output_paths = process_files()   # 这里接住
```

不是返回脚本开头，是返回到调用这行代码的地方。

### 8.11 嵌套 for 循环 = 分组打印

```python
for name in 项目列表:          # 外层：每个项目名
    print(f"【{name}】")        # 打印标题
    for r in records:           # 内层：翻一遍所有记录
        if r["项目"] == name:   # 只挑属于这个项目的
            print(r)            # 缩在标题下面打印
```

**执行效果：** 项目A 的标题 → 项目A 的所有记录 → 项目B 的标题 → 项目B 的所有记录

内层每次都从头到尾扫一遍所有记录，用 `if` 筛选出属于当前组的。

### 8.12 `set()` 自动去重

```python
project_names = set()
project_names.add("A")
project_names.add("B")
project_names.add("A")   # A 已存在，自动忽略
# 结果：{"A", "B"}，只有一个 A
```

`.add()` 往集合里扔，重复的自动 ignore。列表用 `.append()`，集合用 `.add()`。

### 8.13 `sorted()` 排序

不加 `sorted()`：集合里的顺序随机，每次跑可能不一样
加 `sorted()`：按字母/数字固定排好再输出

```python
for name in sorted(project_names):
```

用户界面用 `sorted()` 体验更好，不会跳来跳去。

### 8.14 `lambda x: x["字段"]` 排序公式

```python
列表.sort(key=lambda x: x["用哪个字段来排"])
```

- `lambda` = 一个用完就扔的临时小函数，不用 `def` 起名字
- `key=` 要的是**函数**，不是字符串
- 每个元素会送进这个函数，按函数返回的值比大小

### 8.15 `.endswith()` 判断文件结尾

```python
if f.endswith(".md"):
```
- 返回 True 或 False
- 常用来筛选特定类型的文件

### 8.16 `os.path.expanduser("~")` 自动替换家目录

```python
OBSIDIAN_PATH = os.path.expanduser("~/Documents/知识库/")
# → /Users/用户名/Documents/知识库/
```

不同电脑用户名不同，用 `~` 自动适配，不用改代码。

### 8.17 日期操作三板斧

```python
today = date.today()                       # 今天
monday = today - timedelta(days=today.weekday())   # 本周一
sunday = monday + timedelta(days=6)                # 本周日
```

`.weekday()` 返回值（周一=0 … 周日=6）恰好等于"往前推几天到周一"。

`timedelta(days=N)` 表示 N 天的时间差，日期可以直接加减。

`.date()` 砍掉时间部分只留年月日，方便按天分组。

### 8.18 时间戳转日期

```python
modified_time = datetime.fromtimestamp(os.path.getmtime(file_path))
modified_date = modified_time.date()
```

文件在系统里存的是"一串数字"（时间戳），`fromtimestamp()` 翻译成可读的日期，`.date()` 砍掉具体时间只留日期。
