## 一、函数的本质

```python
def 函数名(参数):
    """做某件事"""
    # 具体步骤
    return 结果
```

- `def` = define，**教会** Python 一个技能
- 函数本身不会执行，需要**调用**（写 `函数名()`）才真干活
- `(参数)` = 告诉使用者"你要用我，得给我材料"
- `return` = 把结果送回给调用处，同时函数结束

### 调用过程（最重要）

```python
第 1 行：result = call_ai_api(content)
                       ↓
           跳进 call_ai_api 函数里面执行
                       ↓
           执行到 return structured_data
                       ↓
           跳回第 1 行，结果存进 result
                       ↓
           继续往下走
```
就像看书时看到"见第 50 页注释"，翻过去看完再翻回来。

## 二、if-else 是二选一，不是先后做

```python
if not os.path.exists("raw_data"):
    os.makedirs("raw_data")
    exit()     ← 程序直接死透，else 永远不会执行
else:
    print("文件夹已存在")
```

关键理解：`if` 只问**一次**条件，真就走 `if`，假就走 `else`。
- 不存在 `if` 干完又去干 `else` 的情况
- `exit()` = 当场断电关机，不是"回到路口重新选"
- 觉得"两条路都走了"是因为**手动运行了两次脚本**

## 三、三组"易混"概念

### 3.1 `import` vs `from ... import`

| 写法 | 使用时 | 场景 |
|---|---|---|
| `import datetime` | `datetime.datetime.now()` | 全拿来，用得少时 |
| `from datetime import datetime` | `datetime.now()` | 只拿一个常用类，省前缀 |

### 3.2 全大写变量（常量）

```python
DATA_FILE = "work_log.json"
```
全大写 = Python 约定：**"这个值固定不变，不要在后面改它"**。好处：只写一次文件名/路径，后面反复用，想改名只改一处。

### 3.3 Python 字典 vs JSON 字符串

```
Python 字典（内存中）              JSON 字符串（硬盘上）
  {"项目": "AI播客"}    dump →     {"项目": "AI播客"}
                        load ←
  用 d["项目"] 取值                 不能取值，报错
  关掉脚本就消失                     文件还在，下次能读回来
```

- `json.dump(数据, 文件)` = 字典 → 写进文件
- `json.load(文件)` = 从文件 → 读回字典
- `json.dumps`（多一个 s）= 转成字符串展示，不存文件

## 四、错误处理 try-except

```python
try:
    with open("文件", "r") as f:
        return json.load(f)
except FileNotFoundError:
    return []   ← 文件不存在也别崩溃，给个空列表
```

- `try` = "试着做下面的事"
- `except 某错误类型` = "如果出了这种错，就这么处理"
- 不处理 → 程序崩溃红字报错；处理了 → 优雅继续跑
- 常见的错误类型：`FileNotFoundError`、`Timeout`、`JSONDecodeError`

## 五、今天学的实用函数

| 函数 | 作用 | 类比 |
|---|---|---|
| `input("提示语")` | 在终端停下来等你打字，回车确认 | 脚本学会聊天了 |
| `os.listdir("文件夹")` | 列出文件夹里所有文件名 | 打开抽屉看有什么 |
| `os.path.getmtime("文件")` | 获取最后修改时间 | 看文件最后一次被改是啥时候 |
| `os.path.getctime("文件")` | 获取创建时间 | 看文件是啥时候建的 |
| `timedelta(days=N)` | 日期加减 | 今天往前/后推 N 天 |
| `sorted(列表)` | 排序 | 按拼音/数字排整齐 |
| `.sort(key=lambda x: x["字段"])` | 按字典里某个字段排序 | 按修改时间排，不按文件名排 |
| `set()` | 集合，自动去重 | 往里加重复的东西，它只保留一个 |

## 六、构建函数的 4 步法（写代码前的思考套路）

```
① 输出：这个函数最后要交出什么？
② 材料：需要什么原材料？
③ 处理：原材料怎么加工成输出？
④ 翻译：按顺序一行行写成代码
```

例：`add_record`
- 输出：往文件里加一条新记录
- 材料：用户通过终端输入项目名、内容、状态
- 处理：打包成字典 → 读旧数据 → 追加 → 存回去
- 翻译：`input()` 三行 → `record = {...}` → `load_records()` → `append()` → `save_records()`
