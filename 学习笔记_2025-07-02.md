## 一、文件操作

### `open()` — 打开文件
- `open("文件名", "r", encoding="utf-8")` = 翻开一本书
- `"r"` 表示只读（不动笔改）
- `encoding="utf-8"` = 用正确的翻译规则读中文，否则乱码

### `f.read()` — 读取内容
- 把文件里所有文字一口气读完，存成字符串

### `f.close()` — 合上文件
- 用完关掉，不然占内存

### `with open(...) as f:` — 自动关门
- Python 帮你自动 `close()`，不用手写
- 缩进结束后文件自动关闭

### `.strip()` — 去掉首尾空白
- `"  你好  \n".strip()` → `"你好"`
- 常用来判断文件是不是"空文件"（只有空格换行也算空）

## 二、数据类型

### `{}` 字典 — 带标签的盒子
```python
person = {"name": "张三", "age": 18}
person["name"]  # 取标签叫 "name" 的值 → "张三"
```
- 靠**关键词（key）** 找**值（value）**，不是按位置找

### `[]` 列表 — 有顺序的篮子
```python
names = ["A", "B", "C"]
names[0]  # 第 1 个 → "A"
names[1]  # 第 2 个 → "B"
```
- 从 **0** 开始数

## 三、三大核心语法

### `for` 循环 — 流水线
```python

    # 列表里每个文件，挨个执行一次
```
- `for 变量 in 列表:` = 取出来 → 干活 → 下一个

```python
if not content.strip():
    continue  # 跳过空文件
```
- `if not 条件` = 如果条件不成立，就……

### `def` 函数 — 造工具
```python
def call_api(text):
    # 把一段代码打包，起个名，以后反复用
    return 结果
```
- `def` = define（定义）
- `return` = 把结果送回来
- 函数名就是最好的注释

## 四、变量 vs 字符串

新手最容易晕的地方：

| 写法 | 意思 |
|---|---|
| `content` | **变量** — 去内存找这个名字的盒子 |
| `"content"` | **字符串** — 就是字母 c-o-n-t-e-n-t 本身 |
| `f"内容：{content}"` | **格式化** — `{}` 里的 content 是变量，替换成真实值 |

**判断方法：带引号就是字符串，不带就是变量。**

## 五、import 是什么

```python
import requests  # 拿来一个工具包
```
- `import 某东西` = 下载并打开一个工具包
- Python 自带的工具不够用，需要从外部"借"工具
- `requests` = 管上网发消息
- `os` = 管操作系统（文件路径、环境变量）
- `json` = 管 JSON 格式转换
- `csv` = 管写 Excel 表格
- `glob` = 管按规则搜文件

## 六、函数与方法

| 写法 | 类型 | 例子 |
|---|---|---|
| `len(x)` | 内置**函数** | `len("你好")` → 2 |
| `x.strip()` | 字符串**方法** | `"  你好  ".strip()` → "你好" |

## 七、调 API（调用大模型）

```python
response = requests.post(API_URL, json=payload, headers=headers)
result = response.json()
ai_text = result["choices"][0]["message"]["content"]
```

| 步骤 | 现实比喻 |
|---|---|
| `requests.post(url, ...)` | 把信寄出去 |
| `response.json()` | 翻译回信（JSON 文本 → Python 字典） |
| 一层层 `["key"]` 剥开 | 拆洋葱，找到真正的正文 |

**为什么拆成这样？** 大模型 API 就是这样设计的，不能改。每步有每步的固定位置。

### 调 API 的固定结构

```python
headers = {                          # 信封信息
    "Authorization": "Bearer 密钥",   # 通行证
    "Content-Type": "application/json",
}
payload = {                          # 信件内容
    "model": "模型名",                # 用哪个模型
    "messages": [                    # 对话内容
        {"role": "system", "content": "设定身份"},
        {"role": "user", "content": "要处理的内容"},
    ],
}
```

### system 和 user 的区别
- **system** = 给模型定身价（"你是资深文案"）
- **user** = 给原材料（"处理这段文字"）

### HTTP headers 是什么
- `Authorization` = 证件（你是谁）
- `Content-Type` = 物品类型（里面装的是什么格式）

## 八、glob 文件搜索

```python
glob.glob("raw_data/*.txt")
```

| 部分 | 意思 |
|---|---|
| `raw_data/` | 限定文件夹 |
| `*` | **通配符**，代表"任意字符" |
| `.txt` | 限定结尾 |

**工作原理：** 走进文件夹，逐个文件检查名字是否匹配规则，把匹配的装进列表返回。

## 九、函数拆分思想（最重要）

```
main()                    ← 总指挥
  ├─ check_environment()   ← 第 1 步：检查环境
  ├─ process_files()       ← 第 2+3 步：遍历 + 处理
  │    ├─ call_api()       ← 对每个文件调 AI
  │    └─ ...             ← 保存文件、记录结果
  └─ save_to_csv()         ← 第 4 步：汇总
```

**分而治之：** 一个大问题拆成多个小问题，每个小问题用一个函数解决。

**设计函数的思路：**
1. 列出整个流程的步骤
2. 每个步骤想清楚**输入**（原材料）和**输出**（成品）
3. 输入输出能接上，就串起来了
