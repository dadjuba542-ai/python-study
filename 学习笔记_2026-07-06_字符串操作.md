# Python 学习笔记 — 字符串操作

## 一、切分与合并

### `.split("符号")` — 按符号切开

```python
"AI播客,AI宝儿,AI视频".split(",")
# → ['AI播客', 'AI宝儿', 'AI视频']
```

- 结果是一个**列表**
- 不传参数时按**连续空白**切（不管多少个空格、换行）

### `"胶水".join(列表)` — 用胶水粘起来

```python
" | ".join(["苹果", "香蕉", "橘子"])
# → '苹果 | 香蕉 | 橘子'
```

- `join` 是 `.split` 的反操作
- 常用于把列表转成 CSV 行：`",".join(tags)`

## 二、查找与判断

| 方法 | 返回 | 找不到时 |
|---|---|---|
| `.find("X")` | 第一次出现的位置（数字） | `-1` |
| `.startswith("X")` | `True` / `False` | — |
| `.endswith("X")` | `True` / `False` | — |
| `.count("X")` | 出现次数（数字） | `0` |

### `.find()` 只返回第一个位置

```python
"苹果香蕉苹果".find("苹果")  → 0
```

找所有位置需要循环来写。

### `True` / `False` 叫布尔值

- 来源：数学家 George **Boole** 的名字音译
- `if` 后面放的都是最终变成 `True / False` 的表达式
- 比较符号：`>`（大于） `<`（小于） `==`（等于） `!=`（不等于） `>=`（大于等于） `<=`（小于等于）

## 三、替换与清理

| 方法 | 作用 | 例子 |
|---|---|---|
| `.replace("旧","新")` | 替换文字 | `"a-b-c".replace("-","/")` → `"a/b/c"` |
| `.strip()` | 去掉首尾全部空白 | `" 你好 ".strip()` → `"你好"` |
| `.removeprefix("X")` | 去掉开头特定文字 | `"https://xxx".removeprefix("https://")` |
| `.removesuffix("X")` | 去掉结尾特定文字 | `"笔记.md".removesuffix(".md")` → `"笔记"` |

### `.strip()` vs `.removesuffix(" ")`

```python
"  你好  ".strip()             → "你好"        # 首尾空白全清
"  你好  ".removesuffix(" ")   → "  你好 "     # 只去掉尾部一个空格
"  你好  ".removeprefix(" ")   → " 你好  "     # 只去掉头部一个空格
```

- `.strip()` 专清空白
- `.removesuffix()` 专去掉特定文字（如文件后缀）

## 四、切片 — 取中间一段

```python
text[起点:终点]     # 从起点取到终点（不含终点）
text[:终点]        # 从开头取到终点
text[起点:]        # 从起点取到结尾
```

- 不是函数，是 Python 自带的语法
- 和 `fruits[0]` 取某一项是同一套机制

## 五、组合实战 — 拆解文件名

```python
f = "周报_2026-07-06_AI播客.md"

# 第 1 步：去掉后缀（从右边切，只切最后一刀）
name = f.rsplit(".", 1)[0]      # → "周报_2026-07-06_AI播客"

# 第 2 步：按分隔符切
parts = name.split("_")         # → ["周报", "2026-07-06", "AI播客"]

# 第 3 步：取各部分
date = parts[1]                 # → "2026-07-06"
project = parts[2]              # → "AI播客"
```

**格式统一很重要。** 文件名规则统一，解析代码才简单。格式不统一，解析代码量翻倍。

## 六、代码解析 vs 大模型

| | 代码硬解析 | 大模型 |
|---|---|---|
| 优点 | 100% 准确 | 格式乱也能处理 |
| 缺点 | 格式一变就崩 | 偶尔幻觉 |
| 适合 | 格式固定、量大 | 语义理解 |

最佳实践：**代码做确定性的事，大模型做需要理解的事。** 两者结合。
