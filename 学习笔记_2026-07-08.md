## 一、装饰器（Decorator）

### 本质
**不修改原函数代码，给它"套一层壳"，加额外功能。**

```python
@装饰器
def 原函数():
    pass
```

等价于：

```python
原函数 = 装饰器(原函数)
```

### 固定格式

```python
def 装饰器名(func):            # func = 被装饰的原始函数
    def wrapper(*args, **kwargs):  # wrapper 接收的参数要和原函数一致
        # 调用前做的事（如打日志、权限检查、打折）
        result = func(*args, **kwargs)  # 调用原函数
        # 调用后做的事
        return result
    return wrapper             # 返回 wrapper 函数本体（不是调用它）
```

### 核心流程（常理不清的地方）

```
你写 @装饰器
  → Python 执行：原函数 = 装饰器(原函数)
  → 原始函数作为 func 参数传进去
  → 装饰器内部造一个新函数 wrapper
  → return wrapper 把 wrapper 交出去
  → 原函数 现在被替换成了 wrapper

你调用 原函数(...)
  → 实际调用的是 wrapper(...)
  → wrapper 先干额外的事，再调 func(...)（原始函数）
  → func 就是那个原始函数在装饰器内部的临时名字
```

### `def` 只是"记住"，不是"执行"

```python
def 函数名():      # 只是记住这个函数的存在，不执行里面代码
    print("hello") # 不会打印

函数名()            # 加上 () 才真正执行
@装饰器             # 装饰器函数在这里才真正执行
```

### 关键：装饰器的 `wrapper` 参数要和原函数保持一致

- 如果装饰器写死了 `wrapper(name, age)`，它只能装饰接收 name 和 age 的函数
- 通用写法用 `*args, **kwargs`，不挑食，适配任意参数的原函数

### 实用例子：动态打折

```python
def apply_discount(func):
    def wrapper(price, level):
        if level == "VIP":
            price *= 0.8
        elif level == "普通":
            price *= 0.95
        return func(price, level)  # 把打折后的价格传给原函数
    return wrapper
```

## 二、递归（Recursion）

### 本质
**函数自己调用自己。** 必须有且只有两个要素：

1. **终止条件** — 不再调自己，直接返回
2. **递推公式** — 调自己，且参数越来越接近终止条件

### 经典例子

| 例子 | 终止条件 | 递推公式 |
|---|---|---|
| 阶乘 `n!` | `n == 1` → 返回 1 | `n * factorial(n-1)` |
| 斐波那契 | `n == 0` → 0，`n == 1` → 1 | `fib(n-1) + fib(n-2)` |
| 遍历文件夹 | 没有子文件夹 | `scan_folder(sub_path)` |

### 递归的坑

1. **重复计算** — 斐波那契不加缓存时，大量重复计算导致极慢
2. **栈溢出** — 递归太深（约 1000 层）会报错
3. **可用循环替代** — 不是所有递归都必要，选可读性更好的写法

### 记忆化优化

把缓存做成装饰器，套在任何递归函数上：

```python
def memoize(func):
    cache = {}
    def wrapper(n):
        if n not in cache:
            cache[n] = func(n)
        return cache[n]
    return wrapper
```

## 三、今日关键认知

1. **`def` 是注册，`()` 才是执行，`@` 也是执行**
   - 先扫完全文件记住所有函数
   - 再逐行执行代码

2. **`func` 在装饰器里就是原函数本人，只是换了名字**
   - `func("小明", 18)` 等价于 `原函数("小明", 18)`
   - `func.__name__` 等价于 `原函数.__name__`

3. **代码行号顺序 ≠ 执行顺序**
   - 第 50 行的 `def` 和第 58 行的 `def` 都是在文件开头就记住了
   - 第 57 行的 `@` 触发装饰器时，两个函数都已存在

4. **写例子要让参数一看就懂**
   - `price, level`（价格、等级）比 `name, age`（名字、年龄）更直观
   - 好的例子减少一半的理解成本
